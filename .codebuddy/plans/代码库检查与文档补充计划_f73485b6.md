## 用户需求

全面检查代码库中的错误、漏洞和逻辑问题，修复后执行项目编译确保构建成功，审查现有项目文档的完整性与准确性，补充缺失的功能描述、API说明和配置指南，确保文档与最终代码保持同步。

## 编译错误发现

### 1. 测试配置错误 (`src/__tests__/setup.ts`)

- 错误: `Cannot find name 'vi'`
- 原因: 未导入 vitest 的全局 `vi` 变量

### 2. App.tsx 中的多个问题

- 第66行: `useVirtualList` 期望1个对象参数但传入了4个位置参数
- 第113行: `scanFolder` 在声明前被使用 (变量提升问题)
- 第202/208行: `updateSettings` 在声明前被使用
- 第509行: `virtualRow` 参数隐式具有 any 类型

### 3. Hooks 类型错误 (`src/hooks/useVirtualList.ts`)

- 类型签名不正确，`useVirtualizer` 的使用方式有问题

## 项目文档分析

现有文档：

- `API_DOCUMENTATION.md` - API接口文档

文档缺失内容：

1. 缺少 `pause_conversion` 命令说明
2. 缺少 `resume_conversion` 命令说明
3. 缺少 `conversion-paused` 事件说明
4. 缺少 `conversion-resumed` 事件说明
5. ConversionProgress 类型缺少性能指标字段说明 (conversion_speed, average_speed, estimated_size, processed_bytes)

## 技术方案

### 修复策略

#### 1. 测试配置修复 (`src/__tests__/setup.ts`)

在 setup.ts 文件中显式导入 vitest 的全局变量：

```typescript
import { vi } from 'vitest';
import '@testing-library/jest-dom';
```

#### 2. App.tsx 修复

- **第66行**: 修改 `useVirtualList` 调用为正确的对象参数格式
- **变量提升问题**: 调整函数定义顺序，将 `scanFolder` 和 `updateSettings` 移动到使用它们的函数之前
- **第509行**: 为 `virtualRow` 添加显式类型注解

#### 3. API文档补充

在 `API_DOCUMENTATION.md` 中补充：

- `pause_conversion` 命令
- `resume_conversion` 命令
- `conversion-paused` 事件
- `conversion-resumed` 事件
- ConversionProgress 性能指标字段

### 技术细节

- 使用 vitest 的 `setupFiles` 配置运行测试初始化
- React Hooks 的调用顺序必须保持一致，不能在条件语句中使用
- TypeScript 严格模式需要显式类型注解
