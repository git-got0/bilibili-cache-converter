## 需求概述

用户需要对B站缓存转换器进行以下改进：

1. **性能优化**

- 进度事件节流（throttle）和防抖（debounce）处理，避免频繁状态更新导致性能问题
- 虚拟滚动技术优化大量文件列表的渲染效率

2. **功能增强**

- 转换任务暂停与恢复控制，支持断点续传和重新开始
- 拖拽选择文件夹功能完善

3. **测试覆盖**

- 核心逻辑单元测试
- 端到端集成测试

4. **监控与日志**

- 引入更详细的性能监控指标
- 完善日志分级记录和输出格式

## 技术栈

- **前端框架**：React + TypeScript + Vite
- **UI组件**：Radix UI + TailwindCSS  
- **测试框架**：Vitest + @testing-library/react
- **虚拟滚动**：@tanstack/react-virtual
- **后端**：Rust + Tauri 2.x
- **日志**：log crate + tauri_plugin_log

## 技术方案

### 1. 进度事件节流/防抖

- **前端**：在 App.tsx 中为 `conversion-progress` 事件监听器添加节流控制，使用自定义 Hook 控制 setProgress 调用频率（推荐 100-200ms 间隔）
- **后端**：在 converter.rs 中优化进度发送逻辑，减少不必要的事件发射频率

### 2. 虚拟滚动

- 安装 `@tanstack/react-virtual` 依赖
- 创建 `useVirtualList` Hook 封装虚拟滚动逻辑
- 重构文件列表渲染区域，使用 useVirtualizer 管理可见项

### 3. 暂停/恢复功能

- **后端修改**：
- 在 AppState 中添加 `is_paused` 状态标志
- 新增 `pause_conversion` 和 `resume_conversion` 命令
- 实现断点续传逻辑（记录已完成文件索引）
- **前端修改**：
- 添加暂停/恢复按钮和状态管理
- 增加 `isPaused` 状态

### 4. 拖拽文件夹支持

- 使用 HTML5 Drag and Drop API
- 在主应用区域添加 dragover/drop 事件处理
- 添加拖拽时的视觉反馈样式

### 5. 测试框架

- 创建 vitest.config.ts 配置文件
- 添加测试依赖：vitest, @testing-library/react, @testing-library/jest-dom
- 在 `src/__tests__/` 目录编写前端测试
- 在 `src-tauri/src/` 目录添加 Rust 单元测试

### 6. 性能监控与日志

- 在 ConversionProgress 中增加性能指标字段（如转换速度）
- 优化日志级别使用（debug/info/warn/error）
- 完善日志格式（时间戳、模块名）
- 配置日志文件输出
